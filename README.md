GitHub Project Feature & Semantic Analysis DatasetThis repository contains a comprehensive dataset of GitHub projects, incorporating structural features, semantic embeddings (via CodeBERT and RoBERTa-large), and LLM-extracted metadata. This data is designed for research into project classification, quality assessment, or developer behavior analysis.📂 Repository StructureThe dataset is organized into the following directories and files:Folder / FileDescriptionreadme/Contains the raw README.md files collected from each individual GitHub project.codeBERT/Semantic analysis and embeddings of the project READMEs generated using the CodeBERT model.roberta-large/Semantic analysis and high-dimensional embeddings generated using the RoBERTa-large model.URLs.xlsxA mapping of project IDs to their original GitHub repository URLs.结构特征.xlsxContains structural metrics (e.g., file counts, directory depth) and the Ground Truth labels for classification.基本特征-GPT.xlsxBasic project metadata, programming language distribution, and key information extracted by GPT (LLM) from the READMEs.📊 Data Details1. Semantic Embeddings (codeBERT/ & roberta-large/)We processed the README files through two state-of-the-art Natural Language Processing models to capture deep semantic meaning:CodeBERT: Optimized for programming contexts and documentation.RoBERTa-large: Provides robust, general-purpose linguistic embeddings.2. Structural Features (结构特征.xlsx)This file includes quantitative data regarding the repository's architecture. Most importantly, it contains the actual classification labels (True Values) used for training and evaluating machine learning models.3. LLM-Assisted Extraction (基本特征-GPT.xlsx)To augment the dataset, we utilized GPT to parse the often unstructured README.md files. This file includes:Project category classification.Primary and secondary programming languages.Key project features and summaries extracted via prompt engineering.🛠 UsageThis dataset is intended for researchers and developers working on:Repository Classification: Predicting the purpose of a repo based on its description and structure.Semantic Search: Training models to understand the relationship between documentation and code.LLM Verification: Comparing human-labeled structural data against LLM-extracted features
